# TOGAF: Ruta de Aprendizaje

Bienvenido a este repositorio dedicado a documentar el proceso de aprendizaje y dominio de TOGAF Standard, Version 10 / 9.2. 

El objetivo es centralizar conceptos clave, diagramas y notas de estudio sobre el marco de trabajo de arquitectura empresarial más utilizado en el mundo.

# Objetivos del Repositorio 

Comprender la estructura del ADM (Architecture Development Method).
Documentar los entregables principales de cada fase.
Servir como guía de preparación para la certificación (Foundations / Practitioner).
Almacenar ejemplos prácticos de artefactos de arquitectura.

# El Ciclo ADM (Resumen Rápido)

El corazón de TOGAF es el Método de Desarrollo de Arquitectura. Aquí se detalla lo que encontrarás en cada sección:

* **Fase Preliminar**: Preparación y personalización del marco.
* **Fase A**: Visión de Arquitectura: Definir el alcance y obtener la validación.
* **Fases B, C, D:** (Arquitecturas de Dominio)
  * B: Negocio.
  * C: Sistemas de Información (Datos y Aplicaciones).
  * D: Tecnología.
* **Fase E:** Oportunidades y Soluciones: Planificación inicial.
* **Fase F**: Planificación de la Migración: Hoja de ruta detallada.
* **Fase G**: Gobierno de la Implementación: Supervisión de la construcción.
* **Fase H**: Gestión de Cambios: Asegurar que la arquitectura evoluciona.
